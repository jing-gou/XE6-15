第一步 · Proposal 1

说出来。
按住说话，松开记好。

📝

创建日程

"明天下午三点到五点开产品会" → 有时间段的事项自动识别。

⏰

创建提醒

"明早七点吃药，每天" → 仅有时间点的事项，支持周期重复。

🔍

即时查询

"今天有哪些安排？""下班要干什么来着？"→ 随时问，即时答。

不是聊天机器人 — 专注记录生活，只记录和提醒，不闲聊。

第二步 · Proposal 2

到点听。
两段式提醒，四件事
一口回应。

🔔

预告（提前 15min）

短铃声 + 语音简要播报。让你心里有个数。

📢

提醒（到时）

语音播报 + 持续铃声。1 分钟无回应则自动推迟。

唤醒词"小鹏小鹏" — 四种响应路径：

"知道了"

结束提醒

"十分钟后再提醒"

推迟提醒

"会议在哪？"

追问详情

"跳舞推迟，其余知道了"

精准边界操作

第三步 · Proposal 3

随口改。
日程不是静态的。

会议室坏了、临时有事、取消某次例会 — 如果说一句话就能改，语音闭环才真正成立。

操作示例确认策略
修改单次"下午会改到四点半"无冲突直接执行
仅改本次实例"周五跑步改周六"无冲突直接执行
永久改周期"以后都改周六"高风险，需显式确认
跳过"周五不去了"单次日程等同取消
删除"删掉每周一早会"必须显式确认
撤销"哎不对，撤销刚才的修改"15min 内有效

关键决策

把一环做到 90 分，
而不是三环摊开到 60 分。

✓ 做✗ 不做
✓时间触发✗位置、蓝牙等非时间触发
✓普通话✗方言、多语言
✓固定唤醒词"小鹏小鹏"✗自定义唤醒词
✓简单周期（每天/每周/每月）✗工作日/节假日规则
✓语音 + IM 回执✗纯 GUI
✓App 前台运行✗后台/锁屏唤醒
